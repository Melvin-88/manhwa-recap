# Покроковий план пілотного запуску (оригінальний контент)

Мета: за ~4 тижні випустити 3–5 відео, перевірити чи пайплайн Memoria дає прийнятну якість/швидкість, отримати перший сигнал з ринку до масштабування.

## Тиждень 1 — Фундамент

**1.1. Формат і ніша (1 день).** Піднішу, суміжну з тим, що вже працює (isekai/regression/"система прокачки"), але як повністю оригінальна серіалізована історія — не переказ конкретного тайтла.

**1.2. Інструменти й акаунти (1–2 дні):**

| Інструмент | Навіщо | Вартість |
|---|---|---|
| Claude (Pro, вже є) | Сценарій/наратив, структура серій | вже оплачено |
| ElevenLabs (Creator plan) | TTS-озвучка укр/рос | ~$22/міс (~100 хв аудіо) |
| Nano Banana 2/Pro (API) | Персонажі/сцени, консистентність | ~$0.03–0.05/зображення |
| Seedream 4.5 (API) | img2img-варіанти, ракурси | ~$0.03/зображення |
| CapCut або ffmpeg-скрипт | Збірка: картинки+аудіо+субтитри | безкоштовно |
| YouTube Audio Library | Музика без копірайту | безкоштовно |
| Canva | Обкладинка, мініатюри | $0–13/міс |

**1.3. Реєстри під нову історію (2–3 дні).** Заповнити `registries/character-registry.json` для 3–5 героїв. Character sheet на кожного: 4–6 референсів (анфас, профіль, зріст, крупний план, 2 емоції) через Nano Banana 2, без тренування LoRA.

Час: 4–6 днів. Вартість: ~$25–35.

## Тиждень 2 — Пілотне відео №1

Прогнати перший епізод через увесь пайплайн (Master Narrative → Scene Intelligence → Storyboard → Registry → Visual Shot Package → Prompt Compiler → Image Director), генератор — Nano Banana 2/Seedream. Озвучити в ElevenLabs (закріпити voice_id). Зібрати відео: Ken Burns на статичних кадрах + автосубтитри (Whisper локально). Обкладинка+назва+опис під SEO ніші.

**Бенчмарк:** якщо повний цикл займає більш ніж 2–3 дні ручної роботи — спрощувати пайплайн перед масштабуванням.

Час: 5–7 днів. Вартість: ~$5–10/відео.

## Тиждень 3 — Відео №2–4, пошук ритму

Повторити цикл 2–3 рази, скорочуючи ручну частину: зафіксувати промпт-шаблони по типах кадру, зафіксувати структуру сценарію, вести таблицю "що спрацювало" по retention.

Час: 7–10 днів. Вартість: ~$15–30.

## Тиждень 4 — Публікація й аналіз

Публікувати з рівним інтервалом. Дивитись retention graph, CTR, джерела трафіку. Telegram-канал одразу, з першого відео.

**Контрольна точка:** відео з утриманням вище ~40–50% → продовжувати масштабувати. Низький retention скрізь → проблема в сценарії/темпі, не в картинках.

## Кошторис пілоту

| Стаття | Вартість |
|---|---|
| ElevenLabs (1 міс) | $22 |
| Генерація зображень (20–30/відео × 4) | $15–25 |
| Canva Pro (опційно) | $0–13 |
| Резерв на правки | $15–20 |
| **Разом** | **~$55–80** |

# Автоматизація через OpenClaw (після пілоту)

OpenClaw — self-hosted gateway для AI-агентів: працює на власному залізі/VPS, підключається до Telegram/Discord/Slack, підтримує tool calling, browser automation, multi-agent routing і **skills** (папка з SKILL.md — формат, використаний у `skills/` цього репозиторію).

## Архітектура

```
Telegram: "нова серія: [тема]"
   ↓
[Агент-Наратор] — skill "master-narrative"
   ↓ структуровані сцени
[Агент-Режисер] — skills "scene-intelligence-engine" + "storyboard-planner"
   ↓ Visual Shot Package
[Агент-Реєстратор] — оновлює registries/*.json
   ↓
[Агент-Генератор] — Nano Banana 2 / Seedream API на кожен шот
   ↓ зображення
[Агент-Озвучувач] — ElevenLabs API, фіксований voice_id
   ↓ аудіо
[Агент-Монтажер] — ffmpeg-скрипт → mp4
   ↓
Telegram: "відео готове, прев'ю" → затвердження
```

**Чому реалістично:** Claude Pro підписка переюзається через OpenClaw (`claude auth login`, Claude CLI reuse — без окремого API-рахунку). Node.js/TS-досвід робить конфігурацію `openclaw.json` і власні skills тривіальними. Реєстри Memoria вже мапляться на формат skills.

**Автоматизувати першим:** механічні виклики TTS і генерації зображень за готовим Shot Package.
**Лишити ручним довше:** сам сценарій і фінальне затвердження відео — автоматичний шаблонний наратив це те, за що YouTube демонетизує.

**Час на побудову:** 1–2 тижні, **після** пілоту, не паралельно з ним.

## Наступна контрольна точка

Після пілоту: робочий сигнал retention/зростання → будувати OpenClaw-автоматизацію для темпу випуску. Слабкий сигнал → ще одна ітерація сценарію/формату перед автоматизацією.
