# Skill: Prompt Compiler

## Purpose
Перетворює Visual Shot Package + дані реєстру в конкретний промпт для обраного генератора.

## Input
- `04-visual-shot-package.json`
- `registries/style-registry.json` для `generator_config` (поле `generation_backend`)

## Output
`05-prompt-package.json`:
```json
{
  "episode_id": "ep01",
  "prompt_packages": [
    {
      "shot_id": "SHOT_001",
      "prompt_text": "",
      "reference_images": [],
      "generator_globals": {},
      "compiler_flags": {}
    }
  ]
}
```

## Rules
- Вирішує конфлікти між registry-даними (напр. якщо освітлення сцени суперечить дефолтному освітленню локації — сцена має пріоритет).
- Будує промпт під `generation_backend` зі `style-registry.json` (`claude-mcp-media` за замовчуванням для пілоту; `fallback_model` — задокументована, неактивна альтернатива, перехід на яку не вимагає змін цього skill, лише перемикання `generation_backend`).

## Формат зображення: багатопанельна композиція
Орієнтир — формат популярних відео-переказів манхви: одне зображення на шот, але **всередині нього 2-3 панелі** манга-сторінки (не одна суцільна ілюстрація), кожна панель — окремий конкретний мікробіт дії чи реакції з `summary`/`emotional_beat` відповідної сцени. Мета — щоб картинка сама несла не менше сюжетної інформації, ніж дикторський текст, а не покладалась на нього.
- **Кількість зображень не змінюється** — досі 1 шот = 1 файл `SHOT_NNN.png`; багатопанельність відбувається всередині цього одного файлу через `prompt_text`, а не через розбиття на нові шоти.
- Промпт явно перелічує 2-3 конкретні мікробіти для окремих панелей (напр. "panel 1: X, panel 2: Y, panel 3: Z") замість одного узагальненого опису дії на весь кадр.
- **Спіч-бабли — лише форма, без тексту всередині.** Якщо в шоті є репліка чи озвучений момент — промпт додає порожній (empty, blank, no text inside) спіч-бабл біля відповідного персонажа, щоб було зрозуміло хто говорить, але не вписує сам текст репліки текстом у зображення (AI-генератори ненадійно рендерять вбудований текст — вихід зазвичай нечитабельна абракадабра).
- **Звукові ефекти (onomatopoeia) дозволені навмисно** в кадрах із різкою фізичною дією (удар, ривок, вибух) — короткий вигук у стилі коміксу (WHOOSH, CLANK, BOOM), явно прописаний у `prompt_text`. Це свідомий елемент стилю, не дефект — Image QA не флагує запитаний SFX-текст (див. правило Image QA), лише незапитаний чи вписаний з помилками.
- Джерело деталей для панелей — те саме, що і для решти пайплайну: `summary`/`emotional_beat` з `01-master-narrative.json`, ніяких нових вигаданих деталей дії.
