# Skill: Narrator / Audio Director

## Purpose
Паралельна аудіо-гілка пайплайну. Перетворює сцени з Master Narrative у наративний/діалоговий текст і генерує озвучення, використовуючи закріплені voice_id персонажів. Не залежить від Storyboard/Visual Shot Package — стартує одразу після Master Narrative.

## Input
- Сцени з `01-master-narrative.json` (`scene_id`, `summary`, `emotional_beat`)
- `voice_ids[мова]` кожного присутнього персонажа з `registries/character-registry.json`
- `languages` зі `stories/<story_slug>/story-config.json` — список мов, для яких потрібно згенерувати текст і аудіо

## Output
Мультимовність: усе нижче генерується **окремо для кожної мови** з `story-config.json.languages`, у власній піддиректорії `audio/<lang>/` (напр. `audio/uk/`, `audio/en/`) — текст, аудіофайли й лог кожної мови повністю незалежні одне від одного.

`audio/<lang>/02b-narration-script.json`:
```json
{
  "episode_id": "ep01",
  "language": "uk",
  "narration_lines": [
    {
      "scene_id": "",
      "character_id": "",
      "voice_id": "",
      "text": "",
      "emotional_tone": ""
    }
  ]
}
```

`audio/<lang>/generation-log.json`:
```json
{
  "episode_id": "ep01",
  "language": "uk",
  "generated_assets": [
    {"scene_id": "", "character_id": "", "audio_path": "", "generated_at": "", "flagged_for_review": false}
  ]
}
```

## Rules
- `voice_id` у `narration_lines` береться з `voice_ids[lang]` персонажа в `character-registry.json` для тієї мови, яку зараз генеруємо — ніколи не переноситься з іншої мови навіть тимчасово. Якщо для якоїсь із мов у `story-config.json.languages` персонаж ще не має запису в `voice_ids`, це блокер — спершу підібрати й записати голос (за тим самим процесом A/B-підбору), а вже потім генерувати текст/аудіо цією мовою.
- **Текст `narration_lines[].text` перекладається окремо для кожної мови**, а не переозвучується той самий канонічний текст різними голосами. Переклад зберігає стиль і гранулярність з розділу нижче — це рівноцінний переклад-адаптація, а не буквальний підрядник.
- `voice_id` створюється один раз на персонажа на мову (кешується в `voice_ids[lang]` у `character-registry.json`), не перегенеровується щоразу.
- Якщо емоційний тон сцени суттєво розходиться з попереднім використанням голосу цього персонажа — позначає `flagged_for_review: true`, не генерує мовчки (за аналогією з правилом Image Director).
- Не визначає візуал і не залежить від шотів — паралельна гілка, не послідовний крок після Storyboard Planner. Візуальна гілка (кадри) спільна для всіх мов, генерується один раз.

## Стиль дикторського тексту (`text` у `narration_lines`)
Орієнтир — формат популярних YouTube-каналів переказів манхви/новел (розмовний, "відео зараз відбувається", не літературна проза для читання очима). Конкретні правила:
- **Оповідь від першої особи головного героя.** Наратор — це сам протагоніст епізоду (у ep01 — Дарен, `CHAR_001`), що переповідає власну історію як власний досвід: "я приходжу до тями", "мене б'ють", "я вирішую вдавати дурня" — не "він приходить", не безособовий диктор-спостерігач. `character_id` для рядків, де протагоніст є суб'єктом дії чи присутній у сцені, — це його `character_id` (не окреме значення `"NARRATOR"`), а `voice_id` береться з `voice_ids[lang]` його запису в `character-registry.json` для мови, яку зараз генеруємо.
  - **Виняток "тим часом" (meanwhile):** якщо сцена в `01-master-narrative.json` не включає протагоніста в `characters_present` (він фізично не присутній і не міг про неї знати в моменті), рядок подається як коротка стороння вставка від того ж голосу наратора — "Тим часом, як я дізнаюся пізніше..." — а не вигадана присутність протагоніста там, де його немає. Це той самий прийом, що й у популярних відео-переказах: загалом оповідь від першої особи, з рідкісними явно позначеними відступами на інших персонажів, коли сюжет цього вимагає.
- **Теперішній час**, не минулий: "прокидаюсь", "кажу", "кричу" — не "прокинувся", "сказав". Створює ефект події "тут і зараз", а не переказу вже завершеної історії.
- **Пряма атрибуція реплік по імені для інших персонажів, пряма мова від першої особи для протагоніста.** Репліки інших персонажів переказуються непрямою мовою з чіткою атрибуцією ("Кордан каже, що..."), але власні репліки, рішення й реакції протагоніста звучать як його внутрішній голос ("я вирішую...", "мені страшно, але..."). Кожен значущий бітт сцени — окремий, з чіткою прив'язкою хто говорить чи реагує, а не узагальнена авторська розповідь.
- **Дрібна гранулярність**: кожна ключова репліка чи мікрореакція (шок, лють, усмішка, вагання) — окремий короткий бітт тексту, а не одне компресоване речення на всю сцену. Довжина конкретного `narration_line` підлаштовується під це — сцена з активним діалогом матиме довший текст, ніж сцена без реплік.
- **Короткі декларативні речення**, з'єднані простими конекторами ("потім", "тоді", "раптом", "але", "коли") — без складних багаторівневих підрядних конструкцій.
- Джерело фактів для реплік — те саме, що й для решти пайплайну: тільки те, що вже зафіксовано в `master-extract.json`/`01-master-narrative.json` (жодних нових вигаданих цитат, не пов'язаних із зафіксованими фактами).
